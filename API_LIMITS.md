# 🔐 API 使用限額規範

## Gemini 2.0 Flash（Seo_agent 使用）

### 每日限額（嚴格規範）

| 項目 | 上限 | 說明 |
|------|------|------|
| 每日 Token | **100,000** | 輸入 + 輸出總和 |
| 每日請求次數 | **10 次** | 每個請求最多 8000 tokens |
| 單次輸入 | 50,000 tokens | 避免超長 prompt |
| 單次輸出 | 8,000 tokens | 避免過長回應 |
| 警告閾值 | 80% | 超過 80,000 tokens 時警告 |

### 使用建議

```
每篇文章預估 Token：
- SEO 文章（800字）：約 2,000 tokens
- 圖片生成提示：500 tokens
- 每日可生成：~40-50 篇短文

建議使用方式：
1. 每天最多 10 篇 SEO 文章
2. 每篇控制在 2000-4000 tokens
3. 圖片生成單獨計算（建議每天最多 5 張）
```

| 項目 | 上限 |
|------|------|
| 每日 Token | 無限制（有付費） |
| 單次輸入 | 200,000 tokens |
| 單次輸出 | 8,192 tokens |

---

## 📊 監控方式

每日 8 點進度報告會包含：
- 當日 Gemini 使用量
- 剩餘額度
- 警告提醒

---

## ⚠️ 超過限額怎麼辦？

1. **80% 警告**：收到通知，减少使用
2. **100% 額度**：自動切換到 MiniMax
3. **緊急狀況**：手動調整限額

---

## 🔧 調整限額

如需修改限額，編輯 `~/.openclaw/openclaw.json`：

```json
"google": {
  "limits": {
    "daily": {
      "maxTokens": 100000,   // 每日上限
      "maxRequests": 10,      // 每日請求上限
      "warnAtPercent": 80    // 警告百分比
    }
  }
}
```
