## 圖片問題修復記錄（2026-02-09）

### 問題描述
Ryan 報告網站圖片無法正常顯示，截圖顯示圖片載入失敗（alt="圖片描述"）。

### 根本原因
1. 很多 Unsplash 圖片 URL 返回 404（Unsplash 刪除了某些圖片）
2. 部分下載的圖片只有 29 bytes，是錯誤頁面
3. 外部 URL 可能被廣告攔截器或網路問題封鎖

### 解決方案
1. **下載所有圖片到本地**：`source/images/` 目錄
2. **清理損壞的圖片**：刪除小於 10KB 的文件
3. **替換失效的圖片**：使用 Picsum 或可靠的 Unsplash 圖片
4. **更新 Markdown 文件**：將 URL 替換為本地路徑 `/images/photo-xxxx.jpg`
5. **重新部署**：確保所有圖片包含在部署中

### 修復結果
- 72 個圖片 URL 替換為本地路徑
- 54 張圖片成功下載並可用
- 部署包含 54 張圖片
- ✅ 圖片顯示問題已解決

### 發布新文章前必須執行的步驟：

1. **下載所有 Unsplash 圖片到本地**
   ```bash
   cd /Users/ryanchiang/.openclaw/workspace/seo-blog/source/images

   ```

2. **驗證圖片是否有效**
   ```bash
   ls -la *.jpg | awk '$5 < 10000 {print "損壞: " $9}'
   ```

3. **清理損壞的圖片**
   ```bash
   rm photo-XXXXX.jpg  # 刪除小於 10KB 的文件
   ```

4. **替換 Markdown 中的 URL**
   ```
   從: ![alt](https://images.unsplash.com/photo-xxxx?w=800&q=80)
   到:   ![alt](/images/photo-xxxx.jpg)
   ```

5. **重新部署**
   ```bash
   ~/.local/bin/vercel --prod --token zlbfstFVS5WcYtpCFgKg3DKD --yes
   ```
